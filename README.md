# Omnibus TOC Extractor

A web app for extracting table of contents data from omnibus epub files. Drop an epub onto the page and get back the omnibus title, total page count, and the title and starting page of each child book — exportable as JSON.

## Usage

Open the app in a browser, drag and drop an omnibus epub onto the drop zone (or click to browse), and the results appear immediately. Click **Export JSON** to download the extracted data.

**Example output (epub with page metadata):**

```json
{
  "omnibus_title": "Bonded Summoner Omnibus: Books 1-3: A Summoner Fantasy LitRPG",
  "total_pages": 1960,
  "page_count_method": "ncx-metadata",
  "child_books": [
    { "title": "Bonded Summoner Book 1", "start_page": 12 },
    { "title": "Bonded Summoner Book 2", "start_page": 623 },
    { "title": "Bonded Summoner Book 3", "start_page": 1226 }
  ]
}
```

**Example output (epub without page metadata — word count estimate):**

```json
{
  "omnibus_title": "Goblin Breeder: Omnibus Edition: The Complete Series Plus New Content",
  "total_pages": 2203,
  "page_count_method": "word-count-estimate",
  "child_books": [
    { "title": "Goblin Breeder",    "start_page": 3    },
    { "title": "Goblin Adventurer", "start_page": 447  },
    { "title": "Goblin Settler",    "start_page": 791  },
    { "title": "Goblin Defender",   "start_page": 1197 },
    { "title": "Goblin Conqueror",  "start_page": 1733 },
    { "title": "Goblin Protector",  "start_page": 2074 }
  ]
}
```

## Running with Docker

```bash
docker compose up --build
```

Then open [http://localhost:5000](http://localhost:5000).

To run on a different port, edit `docker-compose.yml`:

```yaml
ports:
  - "8080:5000"
```

## Running locally

Requires Python 3.12+.

```bash
python3 -m venv .venv
source .venv/bin/activate       # Windows: .venv\Scripts\activate
pip install -r requirements.txt

FLASK_APP=app.main flask run
```

Then open [http://localhost:5000](http://localhost:5000).

## How it works

Epub files are ZIP archives. The parser reads the epub's `toc.ncx` (EPUB2) or nav document (EPUB3) to extract structure and page data without unpacking anything to disk.

**Child book detection** handles two TOC structures found in real-world omnibus epubs:

- **Nested** — top-level NCX entries that contain child entries are treated as child books. This is the most common structure.
- **Flat** — all NCX entries at the same depth. The parser filters out front/back matter and chapter entries, leaving book-boundary entries. If Book 1 has no explicit title entry, its name is derived from the sibling entries (e.g. "Battle Mage 2", "Battle Mage 3" → Book 1 is "Battle Mage").

When multiple child books share the same title (some omnibuses don't differentiate them in the TOC), they are automatically numbered: "The Arcane Lord" → "The Arcane Lord 1", "The Arcane Lord 2", etc.

**Page counts and start pages** use one of two methods, indicated by `page_count_method` in the output:

- `ncx-metadata` — the epub contains explicit page number data (generated by Calibre or similar tools). Page counts and start pages are exact.
- `word-count-estimate` — no page metadata is present. The parser counts words across all spine files and estimates pages at 250 words per page. Start pages are proportional estimates and will not match any specific edition's print page numbers.

## Limitations

- Child book detection works best when the TOC has a clear structure. Omnibuses where every chapter is listed at the top level with no book-level grouping will not produce child book results.
- Only the first root OPF document is parsed. Epubs with multiple renditions are not supported.
